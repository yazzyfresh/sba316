Reflection Questions

1.  What could you have done differently during the planning stages of your project to make the execution easier?

During the planning stages, I could have created a more detailed outline of the website structure before starting development. Planning the pages, features, and layout beforehand would have made the coding process more organized and helped prevent changes later. I also could have planned the JavaScript features earlier, such as the search bar and form validation, so I would have a clearer idea of how each part of the website would connect.

---

2. Were there any requirements that were difficult to implement? What do you think would make them easier to implement in future projects?

One requirement that was challenging was adding interactive features with JavaScript, especially making the search bar work correctly and connecting user input with the webpage. Debugging small issues like matching HTML IDs with JavaScript selectors took time.

In future projects, these requirements would be easier to implement by planning the functionality first, testing smaller pieces of code before adding more features, and keeping the HTML, CSS, and JavaScript files organized from the beginning.

---

3. What would you add to, or change about your application if given more time?

If I had more time, I would add more advanced features to Front Row Notes, such as allowing users to submit and save their own concert reviews permanently, adding artist search filters, including concert dates and locations, and creating a rating system. I would also improve the overall design with more animations and additional pages for music news, festivals, and upcoming concerts.

---

4. Notes for My Future Self

When creating a similar project in the future, I should spend more time planning the structure and features before starting to code. I learned that organizing my files, naming my HTML elements carefully, and keeping my JavaScript connected correctly can prevent many issues later.

I should also remember to test each feature as I build it instead of waiting until the end. Small steps make debugging easier, especially when working with JavaScript and interactive features. Keeping my code clean and adding comments helps me understand my own work when I return to it later.

Most importantly, I learned that building a project takes patience and problem-solving. Errors are part of the process, and using debugging tools like the browser console helps identify and fix problems faster. I should continue practicing by adding more features and improving my projects over time.
